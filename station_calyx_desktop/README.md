# Station Calyx Desktop

> A graphical interface for Station Calyx Ops Reflector.

## Role: Presentation Layer

The desktop app is a **VIEW + TRIGGER** layer only.

- All logic remains in the existing service and API
- UI actions map 1:1 to API calls
- No new intelligence or authority

---

## What This Is

| ? Does | ? Does Not |
|--------|------------|
| Display system status | Execute commands |
| Show trends and advisories | Modify files |
| Trigger existing API actions | Make decisions |
| Present information | Add new capabilities |

---

## Quick Start

### 1. Install Dependencies

```bash
pip install -r station_calyx_desktop/requirements.txt
```

### 2. Start the Service

```bash
python -m station_calyx.ui.cli start
```

### 3. Launch Desktop App

```bash
python -m station_calyx_desktop
```

## System Tray

When minimized, the app shows a tray icon:
- **Green dot**: Service connected
- **Click**: Show/hide window
- **Right-click**: Menu (Show, Quit)

---

## Constraints

### What the Desktop App Cannot Do

1. **No free-text input** - No chat-like interfaces
2. **No "Ask Calyx"** - No conversational UI
3. **No execution controls** - Cannot run commands
4. **No direct file access** - All data via API

### All Actions Map to API

| Button | API Endpoint |
|--------|--------------|
| Capture Snapshot | POST /v1/snapshot |
| Generate Reflection | POST /v1/reflect |
| Generate Advisory | POST /v1/advise |
| Run Temporal Analysis | POST /v1/analyze/temporal |
