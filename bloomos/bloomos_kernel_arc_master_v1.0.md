1. Bloom Gate Layer
1.1 bloom_gate_schema

Defines the canonical structure for all BloomOS gate files.

{
  "version": "string",
  "bloom_status": "string",
  "activation_allowed": "boolean",
  "owner": "string",
  "notes": "string",
  "timestamp": "number"
}


Constraints:

activation_allowed MUST remain false unless Architect explicitly changes it.

bloom_status MUST remain conceptual-only.

Gate MUST never be generated by any agent.

1.2 bloom_gate_lifecycle

Allowed states for bloom_status:

CONCEPTUAL_ONLY ‚Äî default, safe mode, inert

OBSERVATION_ONLY ‚Äî kernel may read, but no activation

READY_FOR_REVIEW ‚Äî Architect-only pre-activation staging

DO_NOT_ACTIVATE ‚Äî hard stop, Architect lock

ARCHIVED ‚Äî past gate stored for historical reference

No state implies activation.
Only an Architect-written gate may change these statuses.
